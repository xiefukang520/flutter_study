# Flutter AI 开发规则（工程化版本）

## 1. 技术栈（强约束）

* Flutter
* Riverpod（StateNotifier）
* Dio
* go_router
* MVVM + Repository

---

## 2. 架构规则（必须遵守）

* UI → 不允许包含业务逻辑
* ViewModel → 处理状态 + 业务逻辑
* Repository → 负责数据获取
* API → 不可直接在 UI 调用

---

## 3. 模板规则（必须按结构生成）

生成代码必须包含：

### UI 层

* ConsumerWidget
* 只负责展示 + 事件

### ViewModel

* StateNotifier
* 提供 loadData / refresh / error handling

### Repository

* API 调用封装
* 返回 Model

---

## 4. 输出规范（强制）

AI 输出必须：

1. 提供完整文件结构
2. 每个文件完整代码
3. 包含 import
4. 可直接运行
5. 不允许伪代码

---

## 5. 禁止规则

* 禁止 setState 管理复杂状态
* 禁止 UI 直接请求 API
* 禁止混用状态管理
* 禁止不完整代码

---
